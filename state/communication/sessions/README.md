# 사람↔엔진 대화 세션 로그

뷰어의 `💬 소통` 탭에서 사람이 파트너 자리에 앉아 한 세션을 끝내면, 그 세션의 전체 로그와
판정이 여기에 `YYYY-MM-DD-HHMMSS-<토큰>.json` 으로 저장된다. 쓰는 주체는 엔진이 아니라
뷰어(I/O 경계)이고, 재현 스크립트는 `state/communication/dialogue.py`(재현 가능한 절반, 합성
파트너)이며 라이브 절반의 엔진은 `src/anima_reborn/dialogue.py::DialogueSession` 이다.

## 왜 이 폴더가 git 에 있는가 — 체리피킹 방지

사람은 seedable 하지 않으므로 각 세션은 **세션당 존재 증명**이고, 사람들 평균은 절대 내지
않는다. 여기서 위험한 것은 운영자가 **파트너인 동시에 실험자**라는 점이다 — 나쁜 세션을 버리고
좋은 것만 남기면 12시드가 조용히 1이 되는 일이 거꾸로 일어난다. 그래서 규칙은:

- **중단·음성·양성 세션을 모두 남긴다.** 완주하지 못한 세션도 판정 `no_session_evidence`
  로 남는다.
- 어떤 보고든 이 폴더의 세션 파일 **개수**를 함께 밝힌다. 고른 하나가 아니라 전부가 증거다.

## 한 파일 안에 무엇이 있나

- `log`: 시행별 원시 기록 — 블록·방향·맥락·귀무 갈래·대상·신호·행동·성공, 그리고 학습
  블록에서의 보상(yoked 는 실제 성공과 분리된 동전). 화면이 아니라 하니스가 채점한
  `success` 가 진실값이다.
- `report`: 방향별(A·B, **절대 평균 내지 않음**) 정확도·순열 p·신호폭/행동폭·귀무
  (day-0·고정엔진(에코)·귀머거리·신호파괴·yoked)·교차방향 일관성(k/2)·최종 판정.

여기 파일은 실제 사람이 돌린 세션만 남긴다 — 스크립트로 만든 합성 세션은 증거가 아니므로
넣지 않는다.
