# 조건부 합성 설계 — 보유한 과거가 현재 쓰기에 작용하는 문맥 게이트 (fable)

핵심 한 줄: **게이트는 규칙이 아니라 히스테리시스다.** 셀 0의 새 쓰기를 원점이 아니라
직전에 보유하던 아날로그 상태에서 시작하고, 쓰기 진폭을 측정된 분지(basin) 장벽 근처로
낮추면, 쌍 j가 뒤집히는지는 (현재 비트, 과거 분지 깊이, WALK 노이즈)의 공동 동역학이
결정한다. `retention.py`가 이미 측정한 물리(작은 jolt는 복원, 큰 jolt는 유한 장벽을
통과)를 재사용하는 것이지 새 비선형성을 심는 것이 아니고, 파이썬 `if`가 아니라 장벽
통과 확률이 문맥을 만든다.

## 1. 메커니즘

현행 `SequenceEngine.turn()`은 시프트 후 `self._cells[0] = self._hold(symbol)` —
`channel_trace`가 `initial=(0.0,)*6`, 진폭 ±0.8로 **원점에서** 새로 쓴다. 과거는 쓰기에
아무 작용도 하지 않는다(녹음기).

추가: `SequenceEngine(context: float = 0.0)`.

- `context == 0.0` (기본): 기존 코드 경로를 그대로 탄다 — 비트 동일
  (`default-stays-exact`). 핀: 시드×턴 격자에서 파라미터 부재 엔진과의 tape 전체 +
  `front_values` 일치 테스트.
- `context > 0.0`: 셀 0을 덮어쓰기 전에 직전 보유값 `prev_values`(아날로그 6-튜플)를
  캡처하고, 쓰기를 `initial=prev_values`, `amplitude=0.8*(1-context)`로 수행. 첫 턴(셀 0
  비어 있음)은 기존 경로 그대로. 시딩은 기존 `seed*100_003 + op` 관례 유지.
- 이행 방식: `dialogue.channel_trace`에 defaulted kwargs `initial=None, amplitude=0.8`을
  추가한다(상수와 TELL/HOLD 봉투가 dialogue.py에 살므로 sequence.py에서 CoupledEngine을
  재조립하지 않는다). 기본값 경로가 기존과 비트 동일함은 기존 test_dialogue 핀 +
  명시 테스트로 고정.

왜 이것이 `if`가 아니라 동역학적 결합인가: context=0에서 드라이브는 어느 분지에서 출발해도
포화 덮어쓰기 → 읽기=현재(현행과 동일 거동). context가 장벽 영역이면, 현재 비트가 과거와
일치하는 쌍은 이미 앉아 있는 분지를 드라이브가 강화하므로 유지되고, 불일치하는 쌍은
장벽을 넘어야 하며 성공 여부는 진폭·과거 분지 깊이·그 시행의 WALK 노이즈의 함수다.
읽기가 (현재, 과거)의 공동 함수가 되는 지점은 코드의 분기가 아니라 장벽 통과 확률이다.

범위 명시: 게이트가 닿는 과거는 정확히 한 턴(셀 0의 직전 보유)뿐이다. 더 깊은 문맥은
게이트의 시간적 합성으로만 생길 수 있고 이번 부품의 주장 밖이다. 시프트와의 상호작용:
셀 1로 내려가는 것은 셀 0의 디코드된 word이므로 게이트를 켜면 문맥-수정된 읽기 r이
내려간다 — 테이프의 망각 곡선은 게이트 켬 조건에서 재측정해 함께 보고한다(4절의 가격).

## 2. 시너지 측정 — 무엇을 읽고 무엇을 계산하나

**read의 정의(모호성 제거):** r_t = `turn(s_t)` 직후의 `tape()[0]` — 표준 HOLD를 거친
셀 0의 차동 부호 디코드 word. 예측 변수는 s_t(현재 심볼)와 s_{t-1}(쓰기 시점에 셀 0이
보유하던 직전 심볼). 시행마다 새 엔진 + iid 균일 심볼(엔진에 학습이 없으므로 '외울'
기회 자체가 없고, 남는 위험은 설계자 배선뿐 — 3절 (e)가 죽인다).

**양:** 상호작용정보 II = I(r; s_t, s_{t-1}) − I(r; s_t) − I(r; s_{t-1}). 부호 포함 보고
(음수 = 중복, 양수 = 시너지).

- 비트별(쌍 j): 2×2×2 정확 범주형 plug-in MI + 측정된 셔플 플로어(`accumulation.py`의
  `_floor` 관례 — 분석 보정이 아니라 널을 잰다). 게이트는 쌍별로 작용하므로 주 신호.
- word-레벨(8×8×8): 플로어가 크므로 플로어 동반, 비트별 합과 대조.
- 교차-비트 II(쌍 j 읽기 × 쌍 k 과거, j≠k): ~플로어여야 함 — 독립성 점검.
- `channel-before-carrier`: 게이트에 도착한 것 먼저 — 쓰기 직전 셀 0 보유 word의
  I(보유; s_{t-1})(입력측 문맥)와 현재 드라이브 3비트를 보고하고, 출력의 생존 분율을 옆에.
- `report-the-rank`: 유효 폭 2^H(r)(실효 word 수)와 비트별 균형을 II 옆에 — r을 두 값으로
  붕괴시키며 II를 버는 게이트를 잡는다.

**예측(측정 아님 — 해석적 형태):** 히스테리시스 채널(일치 → r=cur 확실, 불일치 → r=cur
확률 p)의 비트별 II는 양 극단에서 정확히 0이고(p=1 순수 덮어쓰기, p=0 순수 복사 — 복사는
II의 성질상 자동 0) 중간에서 양수다(p=0.5 이론값 약 +0.12 bits/bit). 따라서 II(context)
곡선은 역U여야 하고, 그 형태 자체가 반증 가능한 시그니처다 — 곡선 전체를 잰다.

## 3. 널 배터리 — 각각 무엇을 죽이나

먼저 측정 자체의 한계를 명시한다: **II는 필요조건이지 충분조건이 아니다.** 설계자가 두
word를 디코드해 `r = cur XOR past`를 배선한 룩업 테이블은 II=+1 bit로 만점을 받는다 —
D7의 재림이 정확히 이 모양이다. 순수 복사는 II=0으로 자동 사망하지만, XOR형 플랜트는
II로는 못 죽인다. 그래서 (c)(e)(f) 라우팅 통제가 하중을 진다.

| 널/통제 | 조작 | 죽이는 가짜 |
|---|---|---|
| (a) 셔플 이력 | r_t를 다른 시행의 s_{t-1}과 짝지어 II 재계산 | 추정기 편향·교란이 만든 유령 시너지 — 플로어의 측정 (`artefact-honesty`) |
| (b) 게이트 오프 | context=0 비트동일 엔진에 같은 측정 | 게이트 아닌 곳에서 온 II — 문맥은 게이트가 산 것임의 증명 |
| (c) 귀머거리 게이트 | 진폭 감소는 그대로 두고 `initial`만 원점으로 | 진폭 변화 단독의 효과(충실도만 낮아진 채널) — 시너지가 보유 상태를 경유함의 증명 (`channel-before-carrier`) |
| (d) 룩업 대조군 | 엔진 아닌 파이썬 함수(복사·XOR)에 같은 심볼쌍 | 죽이는 게 아니라 II 단독의 불충분성을 시연하는 팔 — 복사 II=0, XOR II=+1이 나와야 하고, 그래서 (e)가 평결을 진다 |
| (e) 섭동 라우팅 (하중 최대) | 홀드 중 셀 0의 쌍 j를 반대 분지로 jolt한 뒤 s_t를 게이트 쓰기 | 심볼-로그 레벨의 모든 결정론적 룩업(XOR 포함) — 문맥 효과(어느 쌍이 뒤집히기를 저항하는가)는 물리적으로 보유된(jolt된) 부호를 따라가야 하고, 룩업은 명목 s_{t-1}을 따라간다. 정량: I(r_j; jolt후 보유부호 \| s_t) ≫ 플로어, I(r_j; 명목 s_{t-1} \| s_t, jolt후 보유부호) ≈ 플로어 |
| (f) 깊이 그라데이션 | 직전 홀드 길이(분지 침하 깊이)를 변화 | word-레벨 룩업 — 룩업은 깊이를 못 본다. 불일치 쌍의 플립 확률이 아날로그 깊이에 연속 반응해야 함 |

프롬프트의 "새 랜덤 심볼" 통제는 (a) + 시행마다 iid 신선 심볼로 이행된다 — 이 엔진에는
학습이 없으므로 '외운 규칙'의 실체는 설계자 배선이고, 그것을 죽이는 것은 (e)다.

## 4. 평결 문장과 천장

얻는 문장: "게이트 강도 c*에서, 최신 셀의 읽기는 현재 심볼과 엔진이 보유하던 직전 상태에
공동으로 의존했다 — 상호작용정보 +X bits(셔플 플로어 대비), 게이트 오프·귀머거리
게이트에서 소멸, 그리고 보유 상태를 홀드 중 섭동하면 효과가 심볼 로그가 아니라 섭동된
물리 상태를 따라갔다."

금지 문장: "엔진이 문맥을 이해한다" · "합성 의미가 생겼다" · "기억 용량이 늘었다".

천장의 정확한 위치: **시너지는 같은 3비트 예산 안에서 나온다.** I(r; s_t, s_{t-1}) ≤ 3
bits는 그대로이고, II가 양수가 되는 만큼 현재 충실도 I(r; s_t)가 깎인다 — 문맥의 가격.
이 트레이드(게이트 오프 대비 I(r;s_t) 하락분 vs II 상승분)를 표로 보고한다. 선은 여기다:
'사용되는 워킹 메모리'까지 — 현재 응답이 보유 과거의 측정된 함수임. 언어가 되려면 이
공동 의존이 조합적 심볼 공간 위에서 체계적이어야 하는데(지시·통사), 그 성질은 이 기판
어디에도 없고 주장하지 않는다.
